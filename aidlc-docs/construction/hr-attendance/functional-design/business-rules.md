# ビジネスルール詳細定義

## BR-01: 社員番号フォーマット

- 形式: `EMP-XXX`（XXX は3桁以上の連番、ゼロ埋め）
- 例: EMP-001, EMP-002, ..., EMP-999, EMP-1000
- 正規表現: `^EMP-\d{3,}$`
- ユニーク制約（重複禁止）
- 一度付与した社員番号は変更不可（更新対象外）

---

## BR-02: 打刻ルール

### BR-02-1: 打刻種別
- ClockIn（出勤）と ClockOut（退勤）の2種類のみ

### BR-02-2: 打刻順序
- 1つの AttendanceRecord 内で ClockIn → ClockOut の順序を守る
- ClockIn なしに ClockOut は不可
- ClockIn の Timestamp < ClockOut の Timestamp

### BR-02-3: 二重打刻防止
- 同一 AttendanceRecord に同じ Type の打刻は1回のみ
- 既に ClockIn 済みの AttendanceRecord に再度 ClockIn は不可

### BR-02-4: 複数セット出退勤
- 同一社員・同一日に複数の AttendanceRecord を作成可能
- 各 AttendanceRecord は独立した1セットの出退勤（ClockIn + ClockOut）
- 例: 午前出勤→退勤 / 午後出勤→退勤 = 2つの AttendanceRecord
- 時間帯の重複チェック: 前のセットの ClockOut < 次のセットの ClockIn

---

## BR-03: 論理削除

### BR-03-1: 社員の論理削除
- Employee.IsDeleted = true で論理削除
- 論理削除された社員は一覧・検索から除外（デフォルト）
- 論理削除された社員の勤怠データは保持される
- 論理削除された社員でログインは不可
- 管理者画面から論理削除済み社員の一覧表示は可能（復元機能は将来対応）

### BR-03-2: 部署・勤務区分の削除
- 部署: 物理削除（ただし所属社員がいる場合は削除不可）
- 勤務区分: 物理削除（ただし勤怠レコードで使用中の場合は削除不可）

---

## BR-04: 権限ルール

### BR-04-1: 勤怠打刻
- **User**: 自分自身の打刻のみ可能
- **Admin**: 全社員の打刻が可能（代理打刻）

### BR-04-2: 勤怠データ編集
- **User**: 編集不可（打刻のみ、事後修正は Admin へ依頼）
- **Admin**: 全社員の勤怠データを編集可能

### BR-04-3: 勤怠データ閲覧
- **User**: 自分自身の勤怠のみ閲覧可能
- **Admin**: 全社員の勤怠を閲覧・検索可能

### BR-04-4: マスタ管理
- **User**: 閲覧のみ（社員一覧、部署一覧、勤務区分一覧）
- **Admin**: 全マスタのCRUD操作が可能

### BR-04-5: 月次勤怠サマリー
- **User**: 自分自身の月次サマリーのみ閲覧可能
- **Admin**: 全社員の月次サマリーを閲覧・フィルタリング可能

---

## BR-05: 兼務ルール

- 1社員は1つ以上の部署に所属
- 主所属（IsPrimary = true）は社員ごとに必ず1つ
- 主所属の変更時、旧主所属は自動的に IsPrimary = false
- 最後の所属部署を削除することは不可（最低1部署必須）

---

## BR-06: 勤務区分ルール

### BR-06-1: 日次レコードとの関連
- 各 AttendanceRecord は必ず1つの WorkType を持つ
- デフォルトは「通常勤務」（WorkType.Code = "NORMAL"）

### BR-06-2: 半休の扱い
- 半休（午前）/半休（午後）: IsWorkDay = true
- 半休の場合も1セットの出退勤を記録可能
- 労働時間計算には通常通り ClockIn/ClockOut の差分を使用

---

## BR-07: 月次勤怠サマリー計算ルール

### BR-07-1: 勤務日数
- 対象月内の AttendanceRecord のうち、WorkType.IsWorkDay = true のものをカウント
- 同一日に複数レコードがある場合、その日は1勤務日としてカウント

### BR-07-2: 総労働時間
- 各 WorkSession（ClockIn ～ ClockOut）の Duration を合算
- ClockOut が未記録の Session は計算から除外
- 同一日の複数セットは合算

### BR-07-3: 残業時間
- 日ごとに所定労働時間（パラメータ、デフォルト8時間）を超えた分を残業
- 残業 = Max(0, 日の総労働時間 - 所定労働時間)
- 月次残業 = 各日の残業の合算

### BR-07-4: 有給・欠勤日数
- WorkType.Code = "PAID_LEAVE" のレコード数 = 有給取得日数
- WorkType.Code = "ABSENT" のレコード数 = 欠勤日数

## BR-08: バリデーションルール一覧

| 対象 | ルール | エラーメッセージ |
|------|--------|----------------|
| Employee.EmployeeNumber | `^EMP-\d{3,}$` | 社員番号はEMP-001形式で入力してください |
| Employee.Name | 必須、1-100文字 | 氏名は必須です / 氏名は100文字以内で入力してください |
| Employee.Email | 必須、メール形式、ユニーク | メールアドレスは必須です / メールアドレスの形式が不正です / このメールアドレスは既に使用されています |
| Department.Code | 必須、英数字、1-20文字、ユニーク | 部署コードは必須です / 部署コードは英数字で入力してください |
| Department.Name | 必須、1-100文字 | 部署名は必須です |
| WorkType.Code | 必須、英数字_、1-20文字、ユニーク | コードは必須です |
| WorkType.Name | 必須、1-50文字 | 名称は必須です |
| TimeStampEntry | ClockIn < ClockOut | 退勤時刻は出勤時刻より後にしてください |
| TimeStampEntry | 二重打刻禁止 | 既に出勤（退勤）済みです |
| AttendanceRecord | セット間の時間重複禁止 | 前回の退勤時刻より後に出勤してください |
| EmployeeDepartment | 主所属は1つのみ | 主所属部署は1つのみ設定できます |
| EmployeeDepartment | 最低1部署 | 最低1つの部署に所属する必要があります |
| Department 削除 | 所属社員なし | 所属社員がいるため削除できません |
| WorkType 削除 | 使用中レコードなし | 使用中の勤怠レコードがあるため削除できません |
