# 要件定義書 - 人事・勤怠管理システム

## インテント分析

| 項目 | 内容 |
|------|------|
| **ユーザーリクエスト** | 業務システム（人事・勤怠管理）を構築したい |
| **リクエストタイプ** | 新規プロジェクト (Greenfield) |
| **スコープ推定** | 複数コンポーネント（バックエンド + フロントエンド + DB） |
| **複雑度推定** | 中程度（PoC段階、限定機能） |
| **要件分析深度** | Standard |

---

## プロジェクト概要

社内向けの人事・勤怠管理システムのプロトタイプ（PoC）を構築する。
社内の一般社員（数十名規模）が利用し、まずは動作するものを確認することが目的。

---

## 機能要件

### FR-01: マスタデータ管理（CRUD）
- 社員マスタの登録・参照・更新・削除
- 部署マスタの登録・参照・更新・削除
- 勤務区分マスタ（通常勤務、有給休暇、半休、欠勤等）の管理

### FR-02: 勤怠データ管理
- 日次の出勤・退勤時刻の記録
- 勤怠データの登録・編集

### FR-03: 一覧・検索・フィルタリング
- 社員一覧の表示・検索（名前、部署等）
- 勤怠データの一覧表示
- 期間・部署・社員によるフィルタリング

### FR-04: 月次勤怠サマリー
- 社員ごとの月次勤怠サマリーの表示（勤務日数、総労働時間、残業時間、有給取得日数、欠勤日数、勤務区分別日数内訳）
- 年月・部署によるフィルタリング
- 複数テーブル（社員・部署・勤怠・打刻・勤務区分）を横断した集計
- ※ リッチドメインモデルの実装検証を兼ねる（テーブルと1:1にならないドメインモデルのパターン）

---

## 非機能要件

### NFR-01: 技術スタック
| レイヤー | 技術 |
|----------|------|
| **バックエンド** | .NET (C#) |
| **フロントエンド** | ASP.NET Core MVC（Controller + Razor View）。将来的にBlazorとの併用も検討 |
| **データベース** | SQL Server（Docker - Azure SQL Edge） |
| **認証** | OAuth2 / OpenID Connect |

### NFR-02: デプロイ環境
- ローカル開発環境のみ（PoC段階）
- Docker Compose でDB含め起動できる構成

### NFR-03: パフォーマンス
- PoC段階のため、特定のパフォーマンス目標は設定しない
- 数十名規模の同時利用を想定

### NFR-04: セキュリティ
- PoC段階ではセキュリティ拡張ルール（SECURITY-01〜15）はスキップ
- MVP以降のフェーズで適用予定

---

## ユーザー
- **主な利用者**: 社内の一般社員（数十名規模）
- 自身の勤怠入力、勤怠履歴の確認
- 管理者は全社員の勤怠一覧・検索が可能

---

## スコープ（PoC段階）

### 含まれるもの
- 社員マスタCRUD
- 部署マスタCRUD
- 勤務区分マスタCRUD
- 勤怠データの登録・編集
- 一覧・検索・フィルタリング
- 月次勤怠サマリー（リッチドメインモデル検証）
- OAuth2/OIDC による認証

### 含まれないもの（将来対応）
- 帳票・レポート出力（PDF/Excel）
- ワークフロー・承認フロー
- ダッシュボード・高度な集計表示
- 外部システム連携
- クラウドデプロイ
- セキュリティルール全適用

---

## 開発手法
- **TDD（テスト駆動開発）**: Red → Green → Refactor サイクルで実装
  - Domain層・Application層はテストを先に書き、実装はテストを通すために書く
  - テストフレームワーク: xUnit
  - Application層テストではリポジトリをモック化

---

## 技術的考慮事項
- macOS開発環境のため、SQL ServerはDocker（Azure SQL Edge）で運用
- Entity Framework Core を使用したORマッピング
- ASP.NET Core MVC による従来型のController + Razor Viewパターン
