# セルフレビュー観点: バックエンド（Domain / Application / Infrastructure）

## Domain層

### エンティティ
- [ ] エンティティがビジネスロジックを持っているか（貧血ドメインモデルになっていないか）
- [ ] エンティティの生成はファクトリメソッド経由か（コンストラクタの直接公開を避ける）
- [ ] エンティティの不変条件がコンストラクタ・メソッド内で保証されているか
- [ ] 集約ルートを経由せず子エンティティを直接操作していないか

### 値オブジェクト
- [ ] 値オブジェクトが不変（immutable）か
- [ ] 値の等価比較（Equals / GetHashCode）が正しく実装されているか
- [ ] バリデーションがファクトリメソッド内で行われているか

### リポジトリインターフェース
- [ ] インターフェースが Domain層に定義されているか（Infrastructure層ではない）
- [ ] メソッドシグネチャがドメイン言語を使っているか（SQL用語ではなく）
- [ ] 不必要に汎用的になっていないか（IRepository<T> の過度な抽象化）

### ドメインサービス
- [ ] 単一エンティティに属さないビジネスロジックのみを含むか
- [ ] ドメインサービスが外部依存（DB、HTTP等）を持っていないか

## Application層

### アプリケーションサービス
- [ ] ユースケースのオーケストレーションに徹しているか（ビジネスロジックはDomain層に委譲）
- [ ] トランザクション境界が適切か
- [ ] 権限チェックが漏れていないか
- [ ] 戻り値がDTO（エンティティを直接返していないか）
- [ ] null チェック・存在チェックが適切か（存在しないリソースへのアクセス）

### DTO
- [ ] DTO が必要最小限のプロパティのみ持つか
- [ ] DTO にビジネスロジックが含まれていないか
- [ ] 入力用DTO（Create/Update）と出力用DTO が分離されているか

## Infrastructure層

### EF Core
- [ ] DbSet のエンティティ設定が Fluent API で明示的に定義されているか
- [ ] ナビゲーションプロパティが ICollection<T> を使用しているか（List<T> ではなく）
- [ ] 読み取り専用クエリに AsNoTracking() が適用されているか
- [ ] N+1 問題を防ぐため Include() で関連データを明示的にロードしているか
- [ ] ページネーションが大量データのクエリに適用されているか
- [ ] マイグレーションがバージョン管理されているか

### リポジトリ実装
- [ ] EF Core の非同期メソッド（FirstOrDefaultAsync, ToListAsync, SaveChangesAsync）を使用しているか
- [ ] リポジトリがドメインインターフェースの契約を正しく実装しているか
- [ ] クエリロジックがリポジトリ内に閉じているか（LINQ のリーク防止）
